# CLAUDE.md — Mistery（神笔"码"良黑客松 · 故事赛道）

> 给在本仓库工作的 AI / 协作者的项目说明。先读完这份再动手。

## 一句话

为**成人 ADHD** 设计的"睡前思绪接住器"：用户睡前思绪繁杂、想说的太多而难以入睡，
Mistery 用**语音 + 打字**接住这些碎念，帮他**梳理、拆解、归类**，再以**舒缓的感官化输出**助其入睡。

## 它来自哪个故事（重要语境）

- **故事赛道**：`故事五夜.pdf` → **第二夜｜日常掉线的人 · 成人 ADHD**（剧本第 6–10 页）。
- **主角**：程野，23 岁研究生。有能力、点子多、紧急下产出惊人，但**无法"启动"任务**，
  反复"开始失败" → 不回消息 → 自责，陷入羞耻循环。深夜反而能高强度专注（过度专注）。
- **故事的核心洞察（产品的灵魂，所有功能围绕它）**：
  > "我可能不是缺一个更强的计划表。我缺的是一个能在我**开始不了的时候，把我从第一步接起来**的东西。"
  > "如果一个人每次失败都发生在'**开始之前**'，那产品是不是不应该只奖励他**完成以后**？"
- **赛道任务**：围绕"启动 / 切换 / 组织 / 收尾 / 恢复"任一环节设计，帮他在**低能量、混乱、羞耻**中
  完成**一个可持续的小动作**——不是让他更内疚、更高压、更像别人。
- **本组切入点（见 `头脑风暴.md`）**：把上述落到**睡前场景**——思绪太多→难入睡。
  不做"陪聊"，做"把坠落前的小事接住"的异步、低负担工具。

## 产品核心流程（MVP 主线，务必按此实现"输入→输出"保底链路）

1. **记录（输入）**：语音 + 打字，随手倒出睡前的杂念。低门槛、不评判。
2. **个性化解谜 / 梳理**：调用 LLM 把一团乱麻**拆解、归类**（待办 / 担忧 / 灵感 / 情绪…），
   类比 + 唤起以往记忆（日志），帮用户"看见"而非"解决"。
3. **回复输出（感官）**：以舒缓、低刺激的方式呈现（柔和文案 + 助眠音/呼吸节奏 + 暗色界面），
   目标是**让脑子能关掉标签页**，而不是再给一张待办清单。
4. **反馈训练**：记录有没有帮到入睡，逐步个性化。

## 设计红线（来自故事，违背即跑题，评审会扣分）

- **不要羞辱、不要施压、不要更像别人**。文案永远温柔，绝不"你怎么又这样"。
- **奖励"开始"而非只奖励"完成"**。把一个微小启动动作交给外部世界（如故事里"十分钟后再叫我一次，
  别骂我，就说一句该洗澡了"）。
- **异步、低负担、尊重隐私**。不强制实时、不逼用户填表解释自己。
- **不做又一个"更强的计划表"**。程野已经有 Notion/日历/番茄钟/闹钟——别再加一个会变成新任务的工具。

## 技术栈（已定，面向 Python 熟练 + 第一次做移动端）

> 原则：**不碰原生，做手机浏览器里的 PWA**，纯 Python 起步，先打通"输入→输出"保底。

- **形态**：移动端 = **响应式 Web / PWA**。手机浏览器打开网址 →"添加到主屏幕"即像 App。**不写任何原生代码。**
- **框架**：**Streamlit**（纯 Python 单文件，移动浏览器自适应，最省力的保底路线）。
- **AI**：**Anthropic Python SDK**，模型默认 `claude-opus-4-7`（质量）/ `claude-haiku-4-5-20251001`（快/省）。
  用于：思绪拆解归类、感官化文案生成。**务必开启 prompt caching**（缓存 system / 知识库部分）。
- **语音转写**：保底先**只做打字**；有余力再加 `st.audio_input` 录音 → Whisper（API 或 `faster-whisper`）转文字。
- **存储**：**SQLite**（或最简单的本地 JSON）存日志，够用即可，黑客松不要上数据库服务。
- **助眠感官**：`st.audio` 播放舒缓音 + 暗色主题 + 简单呼吸节奏动画，先做最简版。
- **部署**：**Streamlit Community Cloud**（免费）→ 拿到网址 → 手机打开。

### 升级路线（仅在保底完成且有余力时）
- 想要更好的语音/动画/感官体验：换 **FastAPI（Python 后端）+ 单页 HTML（PWA）**，
  用浏览器自带 **Web Speech API** 做语音（零依赖），前端那部分可让 Claude Code 代写。
- 不建议黑客松期间上 React Native / Flutter（学习成本高，与"简单"目标冲突）。

## 项目管理体系（重要 · 后续一切迭代都按此走）

**核心原则：代码（机器跑的）与方案（人读的）彻底隔离。** 不要把设计文档塞进 `src/`，也不要把代码塞进 `docs/`。

```
heronix/
├── CLAUDE.md            # 项目宪法 + 本管理体系（先读它）
├── README.md            # 快速上手
├── requirements.txt
├── .env.example         # 复制为 .env 填 key（.env 不提交）
├── .gitignore
├── .streamlit/          # 暗色主题（必须留在根目录，Streamlit 从运行目录读）
├── .claude/skills/
│   └── run-mistery/SKILL.md   # 启动&使用说明（可 /run-mistery 调用）；每次迭代后更新
│
├── docs/                # ── 方案与文档，不含代码 ──
│   ├── TODO.md          # 唯一待办源；做完勾掉并记入 CHANGELOG
│   ├── CHANGELOG.md     # 版本变更记录（讲演进、便回滚思路）
│   ├── source/          # 赛题原始材料，只读不改（故事五夜.pdf、头脑风暴.md）
│   └── design/          # 产品/技术方案，按版本：vN-简述.md
│
├── src/                 # ── 代码，不含方案文档 ──
│   ├── app.py           # Streamlit 主入口（四屏：倾倒/梳理/收束/晚安）
│   ├── pipeline.py      # 三段 LLM 管线
│   ├── db.py            # SQLite
│   └── prompts/         # 提示词（与代码强耦合，故归 src；红线语气锁这里）
│
├── assets/              # 静态资源（助眠音 mp3）
└── data/                # 运行时数据 mistery.db（gitignore，不提交）
```

### 迭代规矩（你会出多个版本，照此办）
- **方案分版本、永不覆盖**：每次大改在 `docs/design/` 新建 `v2-*.md`、`v3-*.md`，旧版保留（路演能讲演进，也便回退思路）。
- **代码单一演进、不复制目录**：`src/` 始终是当前版本，**不要** `src-v1/`、`src-v2/`。版本差异写进 `CHANGELOG.md`（用 git 时再打 tag）。
- **TODO 是唯一待办源**：所有"还没做"集中在 `docs/TODO.md`；开工→勾选→完成后在 `CHANGELOG.md` 记一笔"做了什么+为什么"。
- **每次大迭代四件套**：① 新增/更新 `docs/design/vN-*.md` → ② 改 `src/` → ③ 更新 `TODO.md` + `CHANGELOG.md` → ④ **同步 `.claude/skills/run-mistery/SKILL.md`**（启动方式或使用流程有变就更新它，并改顶部「对应版本」行）。
- **原始材料只读**：`docs/source/` 是赛题给的，不改。

- **保底优先**：先把"打字输入 → LLM 拆解归类 → 一段舒缓输出"这条链路在 Streamlit 里跑通，再加语音/感官/存储。
- 写任何用户可见文案前，回到上面的"设计红线"对一遍语气。
- 不确定产品取舍时，回到故事核心洞察：**接住"开始"，别再发清单。**
