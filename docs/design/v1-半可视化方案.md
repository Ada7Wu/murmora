## 核心循环

```
   倒念(语音/打字)  ──►  AI拆解·归类·唤起记忆  ──►  感官化收束助眠  ──►  入睡
        ▲                                                              │
        └──────────────  反馈训练(有没有帮到睡)  ◄─────────────────────┘
```

一句话：睡前**倒念** → AI **接住并梳理** → **感官化收束助眠** → 明天它替你记得。

## 一、四屏体验（手机草图）

### ① 入口 · 把今晚交出来
```
┌─────────────────────────┐
│         🌙 23:47        │
│   今晚，脑子里有点吵?    │
│   不用整理，倒出来就行    │
│  ┌───────────────────┐  │
│  │   🎙 长按 说       │  │  ← 语音(P3)
│  └───────────────────┘  │
│  ┌───────────────────┐  │
│  │   ⌨️  我想打字      │  │  ← 打字(P0)
│  └───────────────────┘  │
│   昨晚你放下了 5 件事 ›  │
└─────────────────────────┘
```

### ② 倾倒 · 不评判的容器
```
┌─────────────────────────┐
│  ‹ 返回         随便说    │
│ ┌─────────────────────┐ │
│ │ 明天要交中期汇报还没 │ │  ← 一个大文本框
│ │ 改完，又怕导师催…    │ │     允许语无伦次
│ └─────────────────────┘ │
│   想到啥说啥，标点都不用  │
│  ┌───────────────────┐  │
│  │     接住它 →       │  │  ← 不叫"提交/分析"
│  └───────────────────┘  │
└─────────────────────────┘
```

### ③ 梳理 · AI 把乱麻接住（关键差异点）
```
┌─────────────────────────┐
│   我帮你接住了，看看 ↓    │
│ ┌─ 📌 我替你记下了 ────┐ │  ← 待办:不是清单,是"已托管"
│ │ · 中期汇报 v2        │ │
│ │ 明早 9:00 我提醒你    │ │  ← 外部接住"启动"
│ └─────────────────────┘ │
│ ┌─ 💭 这个担心，眼熟 ──┐ │  ← 担忧:类比+唤起记忆
│ │ "怕导师催"           │ │
│ │ 上周三你也这样，后来  │ │  ← 链接以往日志
│ │ 凌晨改完其实过了。    │ │
│ └─────────────────────┘ │
│ ┌─ ✨ 留着的好点子 ───┐ │  ← 灵感:存起来,别丢
│ │ · 那盏台灯的想法     │ │
│ └─────────────────────┘ │
│  ┌───────────────────┐  │
│  │  好了，可以睡了 →   │  │
│  └───────────────────┘  │
└─────────────────────────┘
```
> 要点：分类**不是为了让你做事**，而是让你看见它**被托管了**，从而有理由放下。
> 措辞永远是「我替你记下了」，不是「你要做 X」。

## 三、LLM 管线（三段，产品的脑子）

```
原始倒念
   │ ① 解析归类 (haiku)  → {todos,worries,ideas,mood,body}
   ▼
   ② 个性化解谜 (opus)   → 对 worries 用类比/旧日志"缩小化"
   ▼
   ③ 感官化收束 (opus)   → 1~2句温柔收束 + 选助眠音
   ▼
渲染卡片 + 助眠 + 写 SQLite
```
提示词三红线（锁在 `src/prompts/*.txt`）：① 永远温柔禁说教；② 是「接住」不是清单；③ 担忧只给透视感不给方案。

## 四、数据模型（SQLite，3 表）

```
dumps                 cards                      feedback
─────                 ─────                      ─────
id                    id                         id
created_at            dump_id → dumps            dump_id → dumps
raw_text              type (todo/worry/idea)     slept_ok   ← 反馈训练
mood                  content                    note
winddown             reframe (worry 专用)        created_at
audio                resolved (担心是否成真)
```

## 六、关键取舍

- 不做原生 / RN（与「简单」冲突，PWA 够路演）。
- 语音是加分非保底（`audio_input`+Whisper 有时延/依赖坑）。
- **别把「梳理」做成待办清单**——最易跑题、最易被扣分的点。
- API key 只进 `.env` / Secrets，不提交。
