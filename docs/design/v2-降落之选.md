## 一句话 & 口号

> 让每一个停不下来的念头，都有地方降落。
> *Let every restless thought find a place to land.*

Murmora 接住 ADHD 用户睡前的多线程念头（任务/情绪/灵感/担忧/回忆/小电影），
拆解归类成**思维导图**或**夜间日志**，完成「睡前认知闭环」，再用一段**短时降落语音 + 呼吸圈 + 渐暗熄屏**
把注意力从发散思考移到身体锚点——确认「今晚已闭环，剩下的交给明天」，然后**放下手机**。

## 二、八屏全流程（初版与最终版共用这条主线）

```
① 启动      ② 降落岛      ③ 倾倒        ④ AI归位      ⑤ 输出         ⑥ 评分      ⑦ 封存      ⑧ 降落
 立石浮现 → 感性/理性 → 语音/打字 → 线程归位 → 导图⇄日志 → 反馈训练 → 池塘收下 → 呼吸→渐暗→晚安
 2-3s      呼吸双岛    60-180s     5-10s不可跳   一键切换    沉入池底   过渡       2-5min 语音
```

线程六类（PDF）：**任务 🗓 / 情绪 💭 / 灵感 ✨ / 担忧 🌊 / 回忆 🌙 / 小电影**。
通道决定默认输出视图：**感性 → 夜间日志**，**理性 → 思维导图**，可一键互切。

## 三、初版（已实现）：Streamlit 八屏可视化

**目标**：先把八屏闭环、暗色水塘质感、呼吸/渐暗的感官走向**走通看见**，路演能从头点到尾。
**取舍**：内容是 **mock**（PDF 的「11月23日 · 池中之石」示例），**不接真实 AI / 语音 / TTS**。

已落地：
- 八屏路由（`st.session_state.screen`）：启动 → 降落岛 → 倾倒 → AI归位 → 输出 → 评分 → 封存 → 降落。
- **池塘母题**：CSS 立石 + 水波扩散 ripple（多屏复用 `POND`）；星点边框 `STARS`。
- **AI 归位**：呼吸光圈 + 三行「接住了…」逐行浮现（1.4s 间隔，仪式感）。
- **输出双视图**：思维导图（中心立石 + 五分支线程）⇄ 夜间日志（第一人称重述）一键切换；底部内联「贴近度」反馈。
- **评分屏**：贴近度滑条 + 多选「哪部分最贴近」+ 选填「哪里更好」→「沉入池底」。
- **降落屏**：4-1-6 呼吸圈 + 三段引导语 + 极细进度条 →「让屏幕暗下来」触发 **CSS 渐暗 overlay** + 熄屏前留言「晚安」。
- 主题改为**深绿水塘色**（`.streamlit/config.toml` + `.stApp` 径向渐变）。

跑起来：`streamlit run src/app.py`（项目根目录执行），手机同 WiFi 用 `电脑IP:8501`。

> **初版的边界（故意不做的）**：真实 LLM 拆解、语音转写、降落语音音频、数据落库、晨间层。
> 这些是「最终版」要补的，见下。

## 四、最终版（目标）：还原 PDF 想呈现的效果

要从「能点完的可视化」变成「PDF 描述的真实体验」，按维度补：

### A. 感官动效（把 Streamlit 的近似换成真实质感）
- 启动 2-3s **自动**跳转（现在要点一下）；立石淡入 + 水波从中心扩散一圈的精确时序。
- 降落岛 4s 缓慢**呼吸**（透明度 70%↔100%）。
- 倾听态：声音**实时**化作水波/波形（音量驱动）。
- 归位屏停留 5-10s **不可跳过**、完成时光圈闪一次再过渡。
- 降落屏呼吸圈与**音频同步**缩放；最后 60s 屏幕亮度**真实**递减后熄屏；封存=抽屉式合上 + **触觉反馈**。
- ↳ 这些动画/转场在 Streamlit 上吃力，**建议切到 CLAUDE.md 的升级路线：FastAPI + 单页 HTML(PWA) + Web Speech API**。

### B. 真实 AI 管线（替换 mock 内容）
- 倾倒文本 → LLM **六类线程识别**（任务/情绪/灵感/担忧/回忆/小电影）+ 置信度。
- 生成**思维导图结构**（中心主题 + 分支节点，可展开详情）与**夜间日志**（第一人称重述 + 那句「看见」）。
- 生成**明日第一步卡片**（把「启动」交给外部世界，呼应故事红线）。
- **夜间认知扭曲检验**：对「乐观偏差/泡沫念头」温和提示（PDF 深层痛点核心创新），语气守红线。
- 接现有 `src/pipeline.py`（解析/安抚/收束三段）并扩成「线程分类 + 双视图生成」，**开 prompt caching**。

### C. 语音（PDF 默认输入方式）
- 录音 → 转写（Web Speech API 零依赖 / Whisper 兜底）灌入倾倒。
- 30s 静默**中断保护**：「够了吗？我帮你整理一下」[还有要说的][开始整理]。

### D. 降落语音（TTS）
- 2-5min 引导语**音频**，脚本三段：0-60s 确认闭环 → 60-180s 身体感受 → 180-240s 放下手机。
- TTS 合成 + 舒缓背景音（`assets/`），呼吸圈跟音频走。

### E. 反馈训练 & 记忆沉淀
- 评分写库（贴近度 / 哪部分最贴近 / 自由文本）→ 个性化语气与降落脚本。
- **唤起以往记忆**：召回旧 worry/idea 做类比（呼应 v1 的「眼熟」卡片）。

### F. 关怀与晨间层（PDF 长期目标，黑客松可裁）
- **激素周期关怀**：经期前夕等阶段对「夜间冲动/美化倾向」更柔的提示（PDF 目标用户 2）。
- **晨间层**：仅 6:00-12:00 出现的「昨夜回顾」入口 + 记忆沉淀。

## 五、设计红线对照（防跑题，评审会扣分项）

| 红线（来自故事） | v2 怎么守 |
| --- | --- |
| 不羞辱、不施压、不更像别人 | 全程温柔文案；评分用「贴近你了吗」而非打分考核 |
| 奖励「开始」而非只奖励「完成」 | 输出含「明日第一步」，把启动交给外部，不发清单 |
| 异步、低负担、尊重隐私 | 倾倒不评判、可留空；反馈全选填；不逼解释自己 |
| 不做又一个「更强的计划表」 | 终点是**放下手机入睡**，不是再给一张待办 |
