## 1. 背景与定位

### 1.1 来自哪个故事
神笔"码"良黑客松 · 故事赛道 · **第二夜｜日常掉线的人（成人 ADHD）**。
主角**程野**，23 岁研究生：有能力、点子多、紧急下产出惊人，但**无法"启动"任务**，反复"开始失败" → 不回消息 → 自责，陷入羞耻循环；深夜反而能过度专注。

### 1.2 核心洞察（产品的灵魂）
> "我不是缺一个更强的计划表。我缺的是一个能在我**开始不了的时候，把我从第一步接起来**的东西。"
> "如果一个人每次失败都发生在'**开始之前**'，产品是不是不该只奖励他'**完成以后**'？"

### 1.3 目标用户 & 场景
- **用户**：成人 ADHD、睡前思绪繁杂、容易自责的人。
- **场景**：睡前——想和说的太多，脑子像开了太多标签页，难以入睡。
- **差异化**：用户不缺 Notion / 日历 / 番茄钟，缺的是"被接住"。**Mistery 不是又一个待办清单。**

## 2. 产品理念 & 设计红线（团队必须共同遵守）

> 这是评审看的产品灵魂，也是所有文案/交互的底线。**违背即跑题。**

1. **接住"开始"，而非只奖励"完成"**——把一个微小启动动作交给外部世界。
2. **是"我替你记下了 / 接住了"，不是"你要做 X"**——分类是为了让用户*有理由放下*，绝不做成待办清单。
3. **永远温柔**——禁止"你应该 / 你怎么又 / 加油 / 振作 / 想开点"。
4. **担忧只给透视感**（"很多担心后来没那么严重"），不给解决方案、不说教。
5. **异步、低负担、尊重隐私**——不做替代真人关系的陪聊工具，不逼用户填表解释自己。

## 3. 核心体验流程（四屏）

```
   倒念(语音/打字) ─► AI拆解·归类·唤起记忆 ─► 感官化收束助眠 ─► 入睡
        ▲                                                        │
        └────────────  反馈训练(有没有帮到睡)  ◄─────────────────┘
```

| 屏 | 名称 | 用户做什么 | 产品给什么 |
|---|---|---|---|
| ① | 倾倒 | 在大文本框把杂念随手倒出（语无伦次也行） | 一个不评判的容器，按钮叫「接住它」而非"提交" |
| ② | 梳理 | 看 AI 接住的结果 | 📌 待办（已替你记下，明早再说）/ 💭 担忧（温柔缩小化 + 唤起旧记忆）/ ✨ 灵感（留着） |
| ③ | 收束 | 跟着呼吸 | 1~2 句温柔收束语 + 呼吸圈（吸4秒/呼6秒）+ 助眠音 |
| ④ | 晚安 | 放下手机 | 渐暗的晚安屏，"明天的事明天再说" |

> 当前实现里 ②③ 同屏（结果页一次呈现卡片 + 收束 + 呼吸），④ 为独立晚安屏。

界面草图见 `docs/design/v1-半可视化方案.md`。

## 4. 功能清单

### 4.1 ✅ 已实现（v1 / P0 保底）

| 模块 | 功能 | 说明 |

## 5. 迭代方向（候选，未排期）

按故事的五环节（启动 / 切换 / 组织 / 收尾 / 恢复）看，Mistery 现覆盖**收尾 + 恢复**，空白在**启动**（恰是故事最强洞察）。候选方向：

- **A. 战略 · 扩边界**：A1 补「启动支持」（睡前接住的事，明早主动把"第一步"递回）；A2 多模态语音输入；A3 智能启停（睡着自动渐隐，对应"传统音视频无法智能启停"的痛点）。
- **B. 深化核心**：B1 回顾屏 + 相关度召回旧记忆（举一反三复盘）；B2 百科知识库类比；B3 反馈训练 + 生理周期。
- **C. 感官输出**：真实助眠音按情绪选；呼吸引导升级 + 语音播报收束（闭眼可听）；睡眠检测自动停止。
- **D. 工程可达**：配 key / 部署公网 / PWA 收尾。
- **E. 路演叙事**：「程野的一夜」一键演示脚本；传统待办 vs Mistery 对比；微数据看板（本周放下 N 件 / 担心成真率）。

> 时间提醒：路演临近时，**D + E 低成本高 demo 收益**，B1 最贴合头脑风暴叙事，A 类大但风险高。

## 6. 技术方案（面向开发，产品可略读）

### 6.1 技术栈
| 层 | 选型 | 理由 |
|---|---|---|
| 形态 | 响应式 Web / PWA（手机浏览器） | 不碰原生，零额外学习成本 |
| 框架 | Streamlit（纯 Python） | 移动浏览器自适应，最省力 |
| AI | Anthropic SDK：`claude-opus-4-7`（质量）/`claude-haiku-4-5-20251001`（快） | 思绪拆解 + 安抚 + 收束 |
| 存储 | SQLite（本地单文件） | 黑客松不起数据库服务 |
| 部署 | Streamlit Community Cloud（免费，待做） | 公网网址供路演 |

### 6.2 系统架构 & LLM 管线
```
手机浏览器(PWA) ──HTTP──► src/app.py(Streamlit)
                              │  输入 → LLM管线 → 渲染/感官
                              ▼
              SQLite(db.py) · Claude API · Whisper(待做)

LLM 三段：
 原始倒念 ─①解析归类(haiku)─► {todos,worries,ideas,mood,body}
          ─②个性化安抚(opus)─► 对 worries 用类比/旧日志"缩小化"
          ─③感官化收束(opus)─► 1~2句收束语 + 选助眠音
```

### 6.3 数据模型（SQLite，3 表）
- `dumps`：每次倾倒（原文、情绪、收束语、助眠音）
- `cards`：拆出的条目（type=todo/worry/idea、content、reframe、resolved）
- `feedback`：反馈训练（slept_ok、note）——表已建，UI 待做

### 6.4 关键取舍
- 不上 React Native / Flutter（与"简单"冲突，PWA 够路演）。
- 语音是加分非保底（转写有时延/依赖坑）。
- **别把"梳理"做成待办清单**——最易跑题、最易被扣分的点。
- 密钥只进 `.env` / Streamlit Secrets，绝不写进代码或提交。

## 7. 如何体验

```bash
pip install -r requirements.txt
streamlit run src/app.py          # 在项目根目录执行
```
- 浏览器自动打开 `http://localhost:8501`；**没配 key 也能跑**（演示模式）。
- 接真实 AI：`cp .env.example .env` 填入 `ANTHROPIC_API_KEY`。
- 手机同 WiFi：`ipconfig getifaddr en0` 查 IP → 手机开 `http://<IP>:8501` →「添加到主屏幕」。

详见技能 `/run-mistery`。
